# 多游戏门户首页 · 设计方案

> 由产品 / UI / 前端架构三方协同设计后落地。首页定位为**平台级门户**,
> 暗黑像素风(dark pixel arcade)+ 动态效果,承载「多游戏平台」叙事,未来可持续接入新游戏。

## 1. 定位与目标

- **门户(Portal)** = 选游戏;**斗牛大厅(Lobby)** = 在斗牛里选房间/匹配。两者职责分离。
- 首页对**未登录用户公开可浏览**;点 live 游戏时才引导登录(带 `redirect` 回跳)。
- 风格刻意与现有卡通游戏对立:复古街机 + 赛博霓虹,黑底 + 青/品红/黄三色霓虹、CRT 扫描线、像素字体、硬边发光。

## 2. 路由信息架构

```
/                       门户首页(暗黑像素,公开访问)
/games/bullfighting     斗牛大厅(原 / 的内容迁移至此,需登录)
/room/[id] /pve/[id]    斗牛对局/人机(暂留顶层;房间为临时会话,待第二款游戏落地再统一收进 /games/*)
/login /register        支持 ?redirect= 回跳
```

未来新增游戏:在 `app/games/<id>/` 加页面 + 往注册表加一条即可,门户自动渲染。

## 3. 游戏注册表(可扩展核心)

`apps/web/src/lib/games/{types,registry}.ts`。门户消费静态 `GAMES` 列表渲染卡片墙,
**新增游戏 = 往数组追加一条**。关键字段:`status`(live/coming-soon)、`accent`(霓虹主题色)、
`entry`(入口路由,live 必填)、`cover`(同源 SVG,满足 CSP)、`sort`(排序,live 在前)。

当前:抢庄斗牛(live)+ 德州扑克 / 跑得快(coming-soon 占位)。

## 4. 暗黑像素设计令牌

集中在 `apps/web/tailwind.config.ts`,**独立 `pixel-*` / `neon-*` 命名空间,与卡通令牌零冲突**:

- 背景层次:`pixel-void/bg/surface/raised/grid`
- 霓虹:`neon-cyan(#00f0ff)` 主 · `neon-magenta(#ff2e97)` 辅 · `neon-yellow(#ffe600)` 点缀 · `neon-lime`
- 文字:`pixel-text/dim/faint`;边框 `pixel-border`
- 圆角 `pixel: 0`(像素风直角);阴影 `pixel`(硬投影)/ `neon-*`(发光)
- 字体:`font-pixel`(Press Start 2P)/ `font-pixel-body`(VT323),经 `next/font` 自托管(CSP 安全),中文回退黑体

## 5. 动态效果(全部带 `prefers-reduced-motion` 降级)

| 动效                       | 实现                                                              |
| -------------------------- | ----------------------------------------------------------------- |
| 像素星空粒子               | canvas(`PixelBackground`,rAF,后台暂停,密度自适应)                 |
| CRT 扫描线滚动             | `.pixel-scanlines::after` + `crt-scan` keyframes                  |
| 标题 glitch 故障           | `.glitch` 双色 clip 抖动                                          |
| 卡片 hover 霓虹光晕 + 浮起 | framer-motion `whileHover` + Tailwind `group-hover:shadow-neon-*` |
| 扫描光带掠过封面           | `scan-sweep` keyframes(仅 live)                                   |
| 入场 stagger 浮现          | framer-motion `staggerChildren`                                   |
| LIVE 角标脉冲              | `led-pulse` keyframes                                             |

## 6. 主题隔离

门户根节点加 `.theme-pixel` 作用域类(暗色背景/像素字体/`color-scheme: dark`),
**不改 `globals.css` 的 body 全局规则**,卡通游戏页零影响。像素组件类(`.btn-pixel` 等)
与卡通的 `.btn-cartoon` 刻意区分(机械按压 vs 弹性陷入)。

## 7. 组件清单

```
apps/web/src/components/home/
  PixelBackground.tsx   canvas 星空粒子背景
  HomeTopBar.tsx        门户顶栏(像素 Logo + 登录态)
  HomeHero.tsx          glitch 大标题 + 霓虹副标 + 主 CTA
  GameGrid.tsx          消费注册表,stagger 入场
  GameCard.tsx          像素游戏卡(封面/标签/状态/hover 辉光/鉴权跳转)
  PixelButton.tsx       像素按钮
  StatusBadge.tsx       LIVE / SOON 角标
apps/web/src/lib/games/{types,registry,useEnterGame}.ts
apps/web/public/games/{bullfighting,texas,run-fast}.svg   同源像素封面
```

## 8. 登录回跳

`lib/client/redirect.ts` 的 `safeRedirect()` 仅放行站内相对路径(防 `//evil.com` 开放重定向);
`useRequireAuth` 在拦截未登录时带上当前路径,登录后回原页面。
